# BackTradeBot

## Что где лежит

```
main.py              — точка входа, запускает FastAPI сервер
.env                 — твои ключи и настройки (создай из .env.example)
requirements.txt     — зависимости

api/                 — HTTP эндпоинты (/bot/start, /bot/stop, /bot/status)
config/              — загрузка настроек из .env
domain/              — логика и математика стратегии (расчёт сетки, средней цены)
infrastructure/      — клиент Binance API
models/              — модели запросов/ответов
services/            — основная логика бота (цикл, стейт-машина)
tests/               — тесты
```

## Как запустить

**1. Установи зависимости**
```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

**2. Создай `.env`**
```bash
cp .env.example .env
```
Вставь в `.env` свои ключи Binance (тестнет или мейннет) и проверь настройки.

**3. Запусти сервер**
```bash
python main.py
```
Сервер поднимется на `http://localhost:8000`

**4. Управляй ботом через API**
```bash
# Запустить
curl -X POST http://localhost:8000/bot/start

# Статус
curl http://localhost:8000/bot/status

# Остановить
curl -X POST http://localhost:8000/bot/stop
```

Или открой `http://localhost:8000/docs` — там интерактивная документация.

## Тесты

Тесты не требуют `.env`, реального API-ключа или подключения к Binance — всё замокано.

```bash
pytest
```

**Что тестируется и когда срабатывает:**

`tests/test_calculations.py` — чистая математика, без сети и без FastAPI. Запускается мгновенно при любом изменении формул сетки, расчёта средней цены или тейкпрофита.

- Цены сетки всегда убывают и находятся ниже рынка
- Суммарные затраты не превышают депозит при любом числе уровней (2–20) и любом масштабе объёмов
- USDT-аллокация растёт геометрически с заданным коэффициентом
- Цена тейкпрофита всегда выше средней цены покупки
- Все возвращаемые значения — `Decimal`, без утечки `float` в финансовые расчёты

`tests/test_bot_service.py` — логика бота с замоканным Binance. Срабатывает при изменении `services/bot_service.py` или `domain/`.

- Заполненный BUY-ордер → автоматически выставляется SELL по правильной цене
- При новом заполнении старый SELL отменяется перед размещением нового
- Заполненный SELL переводит цикл в статус `COMPLETED`
- Смещение сетки: если рынок ушёл вверх выше порога — все ордера отменяются и сетка пересчитывается
- P&L считается точно: revenue − total_spent − комиссии, накапливается по циклам
- Все статусы Binance (`FILLED`, `PARTIALLY_FILLED`, `CANCELED`, `EXPIRED`, неизвестный) обрабатываются корректно

`tests/test_api.py` — HTTP-эндпоинты с замоканным BotService. Срабатывает при изменении `api/router.py` или `models/api_models.py`.

- `POST /bot/start` возвращает 200 / 409 / 400 / 422 / 502 в зависимости от ситуации
- Невалидное тело запроса (короткий ключ, `ordersCount < 2`) отклоняется на уровне схемы (422)
- Ошибка Binance пробрасывается как 502, а не 500
- Каждый ответ обёрнут в `ApiResponse { success, data, error }`
